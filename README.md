本專案為期末挑戰任務之實作成果，選擇**題目 A：智慧健康日誌與風險評估系統**。系統採用完整的 Frontend → API → DB 三層架構，核心技術統一採用課堂所學之**「決策樹分類 (Decision Tree Classification)」**，針對使用者的睡眠時數、當日步數及心情分數進行多層分支判斷，自動評估並分類出目前的健康風險等級（低/中/高）。

## 🎯 核心考核能力實作說明

### 1. 資料流動的理解 (Data Flow)
本系統建立清晰的資料流動閉環：
- **Frontend：** 使用 HTML5 + JavaScript (Tailwind CSS) 建構直覺的互動介面。表單送出後，透過 `fetch` 非同步呼叫後端 API。
- **API (Backend)：** 採用 Node.js (Express) 架設輕量化伺服器，負責接收前端請求、執行決策樹核心演算法，並提供符合規格的 RESTful API。
- **DB (Database)：** 採用 SQLite 嵌入式資料庫。當後端計算出風險等級後，會將完整數據持久化寫入 `health_logs` 資料表，並即時回傳給前端渲染更新。

### 2. 模型的應用整合 (決策樹核心邏輯)
後端 `calculateRiskTree` 函式嚴格遵循**多層分支判斷結構**，而非單一條件判斷。
- **層次架構：** 先判斷「睡眠時數」是否充足 $\rightarrow$ 再判斷「當日步數」活動量 $\rightarrow$ 最後整合「心情分數」進行灰色地帶細分。
- **特徵落差：** 系統能完美處理如「步數正常但心情極差」等複合型中間情境，確保分類結果嵌入商務邏輯。

## 🗄️ 資料庫與種子資料品質 (Data Quality)

為了確保決策樹分支能切分出有意義的結果，系統內建**「具備特定訊號規律」的 90 天種子資料**（非純隨機數據）：
1. **高風險特徵組 (約25天)：** 睡眠少（4.0-5.5小時）、步數少（1000-3500步）、心情差（1-4分）。
2. **中風險普通組 (約40天)：** 各項數值混合普通。
3. **低風險健康組 (約25天)：** 睡眠足（7.0-9.0小時）、步數多（6000-10000步）、心情好（7-10分）。
*註：在保持區間訊號的前提下，數值皆引入 `Math.random()` 進行自然擾動，使其更貼近真實使用者數據。*

## 🔌 API 端點規格 (API Endpoints)

系統 100% 完整實作說明文件建議之 API 規格：

| 方法 | 路徑 | 用途 |
| :--- | :--- | :--- |
| **GET** | `/api/health-logs` | 取得所有健康日誌紀錄（前端歷程追蹤） |
| **POST** | `/api/health-logs` | 新增一筆健康日誌（即時觸發決策樹計算並寫入 DB） |
| **PUT** | `/api/health-logs/:id` | 修改指定日誌（重新執行決策樹評估並更新） |
| **DELETE** | `/api/health-logs/:id` | 刪除指定日誌紀錄 |
| **GET** | `/api/health-logs/risk` | 獨立端點：依網址參數之決策樹邏輯單純計算並回傳風險等級（不寫入 DB） |
